# 施工事例の追加方法

施工事例セクションは、この `works/` フォルダ内の案件フォルダを GitHub Pages 上で自動読み込みして表示します。

`index.html` を手で編集する必要はありません。

## いちばん簡単な追加方法

デスクトップの `施工写真投入` フォルダへ写真を入れてください。

```text
/Users/yasudashinya/Desktop/施工写真投入/
```

その後、デスクトップの `施工写真をHPに追加.command` をダブルクリックすると、写真が自動で `works/` へ振り分けられます。

- HEIC画像も自動でjpgに変換されます
- 写真1枚ごとに `2026-001`, `2026-002` のような案件フォルダを自動作成します
- 変換後の画像名は `after.jpg` になります
- GitHubへの commit / push まで自動で実行します

## 基本ルール

案件ごとにフォルダを1つ作成し、その中に写真を入れてください。

```text
works/
  2026-001/
    before.jpg
    after.jpg
```

- `before.jpg` と `after.jpg` がある場合: 施工前・施工後として2枚表示
- `after.jpg` のみの場合: 施工後写真のみ表示
- `before.jpg` のみの場合: 施工前写真のみ表示
- `before.jpg` / `after.jpg` が無い場合: フォルダ内の画像をファイル名順に表示

対応画像形式は `jpg`, `jpeg`, `png`, `webp` です。

## 文字情報を入れたい場合

写真だけでも掲載できます。案件名・施工内容・工期・参考価格を表示したい場合は、同じフォルダ内に `info.json` を追加してください。

```json
{
  "name": "松山市 カーポート新設",
  "content": "2台用カーポート新設",
  "period": "約1日",
  "price": "40万〜70万円台"
}
```

`info.json` が無い場合は、写真だけのカードとして表示されます。

## 最小構成

施工後写真だけを追加したい場合は、これだけで表示されます。

```text
works/
  2026-002/
    after.jpg
```

## 文字情報付きの構成

```text
works/
  2026-003/
    before.jpg
    after.jpg
    photo-01.jpg
    info.json
```

## 公開方法

案件フォルダを追加して GitHub に反映すると、GitHub Pages 上の施工事例セクションに自動で表示されます。
